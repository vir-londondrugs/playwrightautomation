# context-pack/

Shared runtime standards loaded by the capability harness alongside every skill.

## Files

| File | Purpose |
|------|---------|
| `system/execution-protocol.md` | Agent session identity, state management, context-limit handling, `_progress.json` spec, and recovery standards. Referenced by all skills that manage session lifecycle or output progress signals. Copied by Stepwise into `./context-pack/system/`. |
| `validation-tools.md` | Deterministic validation tool catalog used by skills with quality-gate steps. Generated by the user and placed in `./context-pack/` (not managed by Stepwise). |

## Standalone Skill Execution

When running a skill **inside a capability** (Stepwise, capability YAML, or orchestrator),
these files are loaded automatically — no action required.

When running a skill **standalone** (direct CLI invocation, ad-hoc agent session, or any
context without a capability harness), you must attach both files explicitly:

```bash
# CLI / agent session — attach to context before activating the skill
context-pack/system/execution-protocol.md
context-pack/validation-tools.md
```

For Stepwise, include them in the capability YAML:

```yaml
context_files:
  - context-pack/system/execution-protocol.md
  - context-pack/validation-tools.md
```

> `.agents/docs/ONBOARDING.md § Troubleshooting` for the detailed standalone-run guide.
